#ServerLesson
## ディレクトリ構造
### ls(LiSt)
- -a 不可視ファイルも表示
- -l タイプや権限、所有者や所有グループなどの詳細情報を表示
- -la -aと-l、どちらの情報も表示  
## ディレクトリ移動
###  pwd(Print Woking Directory)
- 現在のディレクトリの位置を出力する
### cd(Change Directory)
- 指定したディレクトリに、ワーキングディレクトリを変更する
- 記号を指定することで、特定のディレクトリに移動できる
1. ~ ホームディレクトリ
2. / ルートディレクトリ
3. - 移動前に位置していたディレクトリ
4. . ワーキングディレクトリ
5. .. ワーキングディレクトリの一階層上のディレクトリ
6. ../../ ワーキングディレクトリの二階層上のディレクトリ
### 絶対パスと相対パスの違い
- 絶対パス: ルートディレクトリから参照するパス  
  →  /(ルート)から表記
- 相対パス: ワーキングディレクトリから参照するパス  
  → /(ルート)を表記しない
## フォルダ・ファイルの作成
###  mkdir(MaKe DIRectory)
- ワーキングディレクトリに、ディレクトリを作成  
  → Directory1/Directory2のように複数作成も可能  
  → 作成ディレクトリが未作成の場合、-pを指定すると作成可能
### touch
- ワーキングディレクトリにファイルを作成
### &&(AND演算子)
- 左辺の実行成功後に、右辺の処理を実行
### cp(CoPy)
- 同じディレクトリに対して、ファイル or ディレクトリのコピーを行う
- バックアップを取りたい時に使用
- cp -r とつけることで、ディレクトリのコピーも可
### mv(MoVe) 変更元 変更後
- mvはファイルやディレクトリの移動を行う
- 違うファイル名にした場合、名前の変更も可
- → 名前の変更の可否は、指定先に同一のものがあるかで決定
### rm(ReMove) 削除するファイル or ディレクトリ
- ファイルやディレクトリの削除を行う
- ディレクトリの削除を行う場合、-r をつける
- <font color="Red">絶対に行わないこと</font>
- > rm -rf /  
  →  / 以下の全て削除され、PCがただの板になる
## ワイルドカード
- *のように特定の条件にマッチする全てを対象として、操作できる便利機能
1. *(0文字以上の任意の文字)
- *_bk.phpの場合、_bk.phpの部分が一致している対象にコマンドを実行
2. ?(任意の文字を一文字表す)
- ?以外の文字が一致するファイル全てを対象とする
- *との違いは、?の箇所(1文字)の置き換えである点
## その他便利コマンド
### cat(CATnate)
- 指定したファイルの内容をターミナル上に出力
- 指定したファイルをechoで出力する仕組み
- grepコマンドとセットで、ファイル内から特定の文字列を検索する時に使用
### >
- 左辺の実行結果の出力を、右辺で指定したファイルに書き込む
- 既存ファイルを指定した場合、中身を全て上書きするので注意
### >>
- 左辺の実行出力結果を右辺で指定したファイルの末尾に書き込む
- 既存ファイルを指定した場合、末尾にテキストを追加
### vi ファイル名
- vimというエディタをターミナル上で起動
- Linuxの環境にログインしている場合、PC内のVSCode等で開くのが手間がかかる  
  → viを使用すれば、直接ファイルを編集できる
#### ノーマルモード(iで切り替え可能)
- ノーマルモードでは編集不可
- 移動や保存、検索や削除が可能
- ノーマルモード コマンド
1. j(下) k(上) h(左) l(右)に移動
2. :w(保存)
3. :wq(保存して終了)
4. :q!(保存せずに強制終了)
5. u(変更内容をひとつ前に戻す)
6. Ctrl + r(変更内容をひとつ前に進める)
#### インサートモード(escキーで切り替え可能)
- ファイルの編集モード
- 入力したキーがファイルに反映される
#### 注意点
1. 自分がどちらのモードを使用しているか把握すること
2. 半角入力に戻し、Escボタンでノーマルモードに戻る
3. 編集をしたら、必ず:wqで保存
4. 誤操作をしたら、:q!で強制終了→ 編集し直す
